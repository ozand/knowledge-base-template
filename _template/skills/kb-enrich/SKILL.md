# kb-enrich ŌĆö Pi Knowledge Base enrichment skill

Enrich lessons in the KB: add cross-links, citations, regenerate `index.md`,
and update `log.md`. Runs automatically after `kb-capture`, but can be invoked
standalone at any time.

## When to use

- Automatically: called by `kb-capture` after a new lesson is written
- Manually: run `/kb-enrich` after editing existing lessons, or when `index.md`
  is stale, or when you want to add cross-links to older lessons

## Procedure

### Step 1 ŌĆö Load all lessons

Read all files matching `~/.pi/kb/lessons/KB-*.md`. For each file:
- Parse YAML frontmatter (between `---` delimiters)
- Extract: `id`, `category`, `tags`, `error_signatures`, `title`
- Record the file path

Build a lookup map: `{ id ŌåÆ { path, category, tags, error_signatures, title } }`

### Step 2 ŌĆö Cross-link enrichment

For each lesson (or the newly captured lesson if called from `kb-capture`):

1. Read the lesson's `tags` and `category`
2. Find other lessons where:
   - At least one `tag` overlaps, OR
   - Same `category`
3. Check the lesson's `## Related` section ŌĆö identify which cross-links are
   already present (by KB id)
4. For any related lesson NOT already linked, append a new line:
   ```markdown
   - [KB-XXXX ŌĆö Title](KB-XXXX-slug.md)
   ```
   Use a file-relative path (just the filename, since all lessons are in the
   same `lessons/` directory).
5. Update `updated:` date in frontmatter to today's date if changes were made.

**Rules:**
- Never link a lesson to itself
- One link per related lesson ŌĆö do not duplicate
- Only link to lessons returned from the map (do not invent links)
- Do not link from headers, code blocks, or frontmatter

### Step 3 ŌĆö Citations enrichment

For each lesson's `tags`, check if any tag matches a known tool with
documentation. If the `## Citations` section is missing a docs link, append it.

Known doc roots (pattern-match on tag name):
| Tag | Doc URL |
|-----|---------|
| `litellm` | https://docs.litellm.ai/docs/ |
| `pi` | Pi docs: `/home/ozand/.npm/_npx/.../pi-coding-agent/README.md` |
| `git` | https://git-scm.com/docs |
| `github` | https://docs.github.com |
| `python` | https://docs.python.org/3/ |
| `bash` | https://www.gnu.org/software/bash/manual/ |

Add only if not already present in `## Citations`. Do not fetch URLs ŌĆö insert
the known doc root as a reference link.

### Step 4 ŌĆö Regenerate index.md

Write `~/.pi/kb/index.md` from scratch:

```markdown
# Pi Knowledge Base ŌĆö Lesson Index

> Auto-generated by kb-enrich. Edit lessons directly, not this file.
> Last updated: YYYY-MM-DD

## pi-auth

- [KB-0001 ŌĆö LiteLLM proxy returns 401 Unauthorized](lessons/KB-0001-litellm-401-auth.md) `high`

## pi-model

- [KB-0002 ŌĆö Pi model not found](lessons/KB-0002-model-not-found.md) `high`

---
*N lessons across M categories*
```

Group by `category` (alphabetical). Within each group, sort by `id`.
Severity badge inline: `\`high\``, `\`critical\``, `\`medium\``, `\`low\``.

### Step 5 ŌĆö Update log.md

Append to `~/.pi/kb/log.md` (create if missing):

```markdown
## YYYY-MM-DD

* **Addition**: [KB-XXXX ŌĆö Title](lessons/KB-XXXX-slug.md)
```

If called from `kb-capture` with a specific lesson id, log only that lesson.
If called standalone, log "Enrichment run" with date and lesson count.

## Notes

- Always preserve existing frontmatter fields ŌĆö only add/update `updated:` date
- If `## Related` section is missing entirely, create it before appending links
- If `## Citations` section is missing, create it before appending links
- `index.md` and `log.md` are generated files ŌĆö users should not edit them
  manually (edits will be overwritten on next enrich run)
- Enrichment is idempotent: running twice produces the same result

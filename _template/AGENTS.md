# Knowledge Base Agents Guide

This knowledge base uses the OKF (Open Knowledge Format) lesson format.

## Agent instructions

### Adding lessons

When you encounter a new non-obvious error:
1. Use `/kb-capture` to record it
2. The skill will write the lesson, run enrichment, rebuild the visualizer, and push to git

### Looking up lessons

When an error occurs, use `/kb-lookup` before retrying.
The skill matches `error_signatures` from `index.yaml` against the error text.

### Enrichment

`/kb-enrich` adds cross-links between related lessons, citations, and regenerates `index.md`.
Runs automatically after `/kb-capture`, but can be invoked standalone.

### Visualization

`/kb-visualize` rebuilds `viz.html` — an interactive graph of all lessons.
Open `viz.html` directly in a browser (no server needed).

### Sync

`/kb-sync` pushes all pending commits to the remote repo and pulls any remote changes.

## File conventions

- Lessons: `lessons/KB-XXXX-<slug>.md` — Markdown with YAML frontmatter
- Schema: `SCHEMA.yaml` — format specification and `_template` block
- Index: `index.yaml` — machine-readable index (for kb-lookup)
- Index: `index.md` — human-readable index (generated by kb-enrich)
- Log: `log.md` — chronological change journal (generated)
- Viz: `viz.html` — interactive graph (generated)
- Manifest: `kb.yaml` — installation and sync configuration

## Do not edit manually

- `index.md` — regenerated by `kb-enrich`
- `viz.html` — regenerated by `generate-viz.sh`
- `log.md` — appended by `kb-capture` / `kb-enrich`
